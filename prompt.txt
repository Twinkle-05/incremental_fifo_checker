Complete the missing RTL in sources/generator.sv and sources/pattern_checker.sv.

Do not modify fifo_top or fifo_sync.

The design is an LFSR-based FIFO data checker. The generator produces an 8-bit pseudo-random byte stream, packs every 8 consecutive bytes into one 64-bit FIFO word, and writes that word into the FIFO. The checker independently recreates the same byte stream based only on FIFO read activity and compares each FIFO output word against the expected value.

Requirements:

Use the same 8-bit LFSR polynomial in both modules.
Reset both generator and checker to DEFAULT_SEED.
On load_seed, load seed; if seed == 8'h00, use DEFAULT_SEED.
Generator advances only when en == 1 and almost_full == 0.
During stalls, generator must hold LFSR state, byte index, and partially packed word.
Pack bytes in generation order into the 64-bit word, with the earliest byte in the least-significant byte position.
Assert wr for one cycle only when the 8th byte completes a word.
Checker asserts rd only when full_empty == 0.
Checker advances its expected LFSR sequence only when it performs a valid FIFO read.
Checker must not use any direct generator state or extra synchronization signal.
For each valid FIFO output, set out = 1 on match; otherwise set out = 0, assert error_flag, and increment error_count.
The implementation must remain correct across enable disable/re-enable periods and FIFO empty/full stalls.
Use synthesizable Verilog/SystemVerilog only.
Preserve all existing module interfaces.
