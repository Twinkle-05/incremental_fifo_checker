Complete the missing RTL implementation for the generator and pattern_checker modules.

The fifo_top and fifo_sync modules are complete and must not be modified. Implement only the missing RTL logic while preserving all existing module interfaces.

Requirements:

Generate an 8-bit LFSR-based pseudo-random byte sequence.
Use the same LFSR polynomial and initial seed in both the generator and the checker.
Pack eight consecutive generated bytes into one 64-bit FIFO word, where the first generated byte occupies bits [7:0] and the eighth generated byte occupies bits [63:56].
The generator shall issue a FIFO write only after all eight bytes have been assembled.
The generator shall advance its LFSR state only when en is asserted and almost_full is deasserted. During stalls, it shall preserve its LFSR state, byte position, partially assembled word, and write state.
The checker shall independently regenerate the expected sequence using only the signals available at its module interface. No internal state, counter, seed update, valid signal, or synchronization information shall be shared from the generator.
The checker shall perform one comparison for each valid FIFO output word and shall advance its expected sequence exactly once for each successful comparison opportunity. If no valid FIFO output word is available, the checker shall preserve its current sequence state.
The generator and checker shall remain synchronized solely through the ordering of FIFO data transfers, including across stalls, idle cycles, and back-pressure.
Assert out when the current FIFO output matches the expected data; otherwise deassert out.
Preserve all existing module interfaces.
Use synthesizable Verilog/SystemVerilog only.
